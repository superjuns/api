# Supabase 自托管部署指南

**域名规划：** `api.cn07.cn`（推荐）  
系统要求：Ubuntu 20.04 / 22.04 / 24.04  
最低配置：**2 核 4GB 内存 + 50GB SSD**（推荐 4核8G）

---

## 一、部署前准备

### 1. DNS 解析

在域名 DNS 管理中添加一条 A 记录：

| 主机记录 | 记录类型 | 记录值 |
|---------|---------|--------|
| `api` | A | 你的服务器公网 IP |

> 等 DNS 生效后再部署 HTTPS，否则证书申请会失败

### 2. 服务器安全组/防火墙放行

| 端口 | 用途 |
|------|------|
| 80 | HTTP（申请 SSL 证书用） |
| 443 | HTTPS（实际访问用） |
| 22 | SSH（必须，不然连不上服务器） |

> 8000 端口不需要对外放行，通过 Nginx 反向代理走 443

### 3. SSH 登录服务器

```bash
ssh root@你的服务器IP
```

## 四、精简版组件说明

默认只启用核心组件，减少资源占用：

| 组件 | 状态 | 用途 |
|------|------|------|
| db (PostgreSQL) | ✅ 启用 | 数据库 |
| postgREST (REST) | ✅ 启用 | REST API 接口 |
| Kong (API 网关) | ✅ 启用 | API 统一入口 |
| Auth (认证) | ✅ 启用 | 用户认证 |
| Studio (管理后台) | ✅ 启用 | 可视化管理 |
| Meta (元数据) | ✅ 启用 | Studio 依赖 |
| Realtime | ❌ 禁用 | 实时订阅（游戏不需要） |
| Storage | ❌ 禁用 | 对象存储（游戏不需要） |
| imgproxy | ❌ 禁用 | 图片处理 |
| Edge Functions | ❌ 禁用 | 边缘函数 |
| Analytics | ❌ 禁用 | 日志分析 |
| Vector | ❌ 禁用 | 向量搜索 |

### 如何启用某个禁用的组件

编辑 `/opt/supabase/docker-compose.slim.yml`，删除对应服务的 `profiles` 配置，然后重启：

```bash
cd /opt/supabase
docker compose up -d
```

## 五、从 wzkjgz 迁移到 Supabase

### Step 1：创建表结构

打开 Studio → SQL Editor，执行建表语句。

**示例：白名单表**
```sql
create table white_list (
  id bigint generated by default as identity primary key,
  name text not null,
  class text not null,
  created_at timestamp with time zone default timezone('utc'::text, now()) not null
);

-- 开启行级安全
alter table white_list enable row level security;

-- 允许公开读取
create policy "Public read" on white_list for select using (true);

-- 允许公开插入
create policy "Public insert" on white_list for insert with check (true);
```

**示例：游戏成绩表**
```sql
create table game_scores (
  id bigint generated by default as identity primary key,
  game_name text not null,
  student_name text not null,
  class text not null,
  score integer not null,
  total_time integer,          -- 总用时（秒）
  wrong_questions jsonb,       -- 错题数据
  created_at timestamp with time zone default timezone('utc'::text, now()) not null
);

alter table game_scores enable row level security;
create policy "Public read" on game_scores for select using (true);
create policy "Public insert" on game_scores for insert with check (true);

-- 建索引，加速查询
create index idx_game_scores_game_name on game_scores(game_name);
create index idx_game_scores_created_at on game_scores(created_at desc);
```

### Step 2：迁移数据

**方式 A：Studio 导入（简单）**
1. 从 wzkjgz 导出 Excel/CSV
2. Studio → Table Editor → 对应表 → Import data
3. 上传文件，字段映射一下就行

**方式 B：脚本批量插入（量大时用）**
写一个 Node.js/Python 脚本，读旧数据 → 调 Supabase API 写入。

## 六、游戏代码改造

把原来的 wzkjgz API 调用改成 Supabase 调用。

### 方式一：直接用 REST API（改动最小）

Supabase 的 REST API 和 wzkjgz 的用法类似，都是 HTTP 请求：

```javascript
// 读取排行榜
const SUPABASE_URL = 'https://api.cn07.cn/rest/v1';
const ANON_KEY = '你的anon_key';

fetch(`${SUPABASE_URL}/game_scores?select=*&order=score.desc&limit=10`, {
  headers: {
    'apikey': ANON_KEY,
    'Authorization': `Bearer ${ANON_KEY}`
  }
})
.then(res => res.json())
.then(data => {
  // data 就是排行榜数据
});
```

### 方式二：用 Supabase JS SDK（推荐，长期更好用）

```html
<script src="https://cdn.jsdelivr.net/npm/@supabase/supabase-js@2"></script>
<script>
  const supabase = window.supabase.createClient(
    'https://api.cn07.cn',
    '你的anon_key'
  );

  // 读取排行榜
  const { data, error } = await supabase
    .from('game_scores')
    .select('*')
    .order('score', { ascending: false })
    .limit(10);

  // 提交成绩
  const { data, error } = await supabase
    .from('game_scores')
    .insert({
      game_name: 'chengfa-yure',
      student_name: '张三',
      class: '1班',
      score: 100
    });
</script>
```

## 七、数据备份与恢复

### 自动备份

部署时已配置，每天凌晨 3 点自动备份，保留 7 天。

备份位置：`/opt/supabase/backups/`

### 手动备份

```bash
bash /opt/supabase/backup.sh
```

### 恢复备份

```bash
# 解压
gunzip backup_file.sql.gz

# 恢复（注意：会覆盖现有数据！）
cat backup_file.sql | docker exec -i supabase-db psql -U postgres postgres
```

## 九、常见问题

### Q: 服务启动了但访问不了？
1. 检查安全组/防火墙是否放行 80 和 443 端口
2. `docker compose ps` 看服务是不是 healthy
3. `docker compose logs` 看有没有报错

### Q: HTTPS 证书申请失败？
确认 DNS 解析是否生效：
```bash
nslookup api.cn07.cn
```
如果能解析到服务器 IP 但还是失败，等几分钟再试。

### Q: 内存不够用？
- 精简版已经关掉了 6 个非必要服务
- 如果还是不够，考虑升级服务器配置
- 或者进一步禁用 auth、meta 等服务（但 Studio 可能用不了）

### Q: 忘记 Dashboard 密码？
```bash
grep DASHBOARD_PASSWORD /opt/supabase/.env
```

### Q: 数据库连接不上？
默认数据库只在容器内部暴露，不对外。
如果需要外部连接，编辑 `docker-compose.yml` 中 db 的 ports 配置。
